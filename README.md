# 操作系统实验报告 LaTeX 模板使用说明

本项目提供一份可直接使用的操作系统实验报告模板，适合天津大学操作系统原理课程实验报告场景。你只需要改少量内容，就能快速生成排版规范的 PDF 报告。

如果你完全没接触过 LaTeX，也可以从本文档一步一步完成。

## 1. 项目文件说明

当前核心文件如下：

- [main.tex](main.tex)：报告源文件（你主要编辑这个）
- [main.pdf](main.pdf)：已编译的示例 PDF
- [TJU.png](TJU.png)：封面校徽图片
- [Example.jpg](Example.jpg)：正文示例图片

编译过程中会自动生成一些中间文件（如 `.aux`、`.log`、`.toc` 等），这些通常不需要手动修改。

## 2. 最快上手（推荐新手）

如果你不想配置本地环境，建议直接用 Overleaf：

1. 打开 Overleaf 并新建项目（上传本项目所有文件）。
2. 在文件列表中打开 [main.tex](main.tex)。
3. 点击菜单设置编译器为 `XeLaTeX`。
4. 点击 Recompile 编译。
5. 修改封面信息和正文内容后再次编译，下载 PDF。

为什么是 XeLaTeX：

- 模板使用 `ctexart`，对中文支持更稳定。
- 若使用 `pdfLaTeX`，通常会出现中文或字体相关报错。

## 3. 本地使用（Windows）

如果你希望离线写作，按下面步骤进行。

### 3.1 安装 TeX 发行版

Windows 常见两种方案，任选其一：

- 方案 A：安装 TeX Live（功能全，体积较大）
- 方案 B：安装 MiKTeX（按需安装宏包，体积较小）

安装完成后，确保命令行可识别 `xelatex` 或 `latexmk`。

### 3.2 安装编辑器（任选）

- TeXstudio（对 LaTeX 友好）
- VS Code + LaTeX Workshop 扩展

### 3.3 编译命令

在项目目录打开终端，执行：

```bash
latexmk -xelatex main.tex
```

编译成功后会生成/更新 [main.pdf](main.pdf)。

如果你没有 `latexmk`，也可以先用：

```bash
xelatex main.tex
xelatex main.tex
```

说明：第二次编译用于刷新目录、交叉引用等内容。

## 4. 你需要修改哪些内容

主要编辑 [main.tex](main.tex)，推荐按下面顺序改：

1. 封面信息
2. 目录后的正文章节
3. 代码块与实验截图
4. 结论与心得

### 4.1 封面信息

在封面区域把示例文字替换为你的信息，例如：

- 题目
- 学院名称
- 专业
- 学生姓名
- 学号
- 年级
- 班级
- 时间

### 4.2 正文结构

模板已提供典型实验报告结构：

- 实验名称
- 实验目的
- 实验内容
- 实验步骤
- 实验分析
- 实验结论及心得体会

你可以直接在对应章节替换示例内容。

### 4.3 插入代码

模板已配置 `listings` 代码高亮。直接保留 `lstlisting` 环境并替换代码即可。

如果你写的不是 C 语言，可以把 `language=C` 改成对应语言（如 `language=Python`）。

### 4.4 插入图片

把图片放在项目目录，再在 [main.tex](main.tex) 中修改文件名，例如：

```tex
\includegraphics[width=0.6\textwidth]{你的图片文件名.png}
```

注意：文件名区分大小写，建议只用英文字母、数字和下划线。

## 5. 常见问题与解决

### 5.1 报错：找不到图片文件

现象：`File 'xxx' not found`

排查：

1. 确认图片在项目目录内。
2. 确认 [main.tex](main.tex) 中的文件名与实际一致（包含后缀）。
3. 避免中文文件名和空格。

### 5.2 报错：中文乱码或字体异常

排查：

1. 编译器必须是 `XeLaTeX`。
2. 不要改成 `pdfLaTeX`。
3. 若本地仍异常，优先在 Overleaf 测试确认是否为本地环境问题。

### 5.3 目录不更新

解决：至少编译两次，或使用 `latexmk -xelatex main.tex` 自动处理。

### 5.4 页面警告（如 overfull/underfull）

这类通常是排版警告，不一定影响最终提交。优先检查是否有超长英文单词、超宽代码行、超大图片。
